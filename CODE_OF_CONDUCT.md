# Code Of Conduct

## Our Standard

This project expects respectful, professional participation.

Examples of acceptable behavior:
- asking focused technical questions
- providing constructive feedback
- reporting bugs and security issues responsibly
- assuming good intent while discussing design or implementation tradeoffs

Examples of unacceptable behavior:
- harassment, insults, or personal attacks
- discrimination or hateful conduct
- intentionally disruptive behavior
- publishing private information without consent
- posting secrets, API keys, or other sensitive material

## Scope

This code of conduct applies to repository discussions, issues, pull requests, security reports, and other project communication channels.

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, and other contributions that violate this code of conduct.

## Reporting

For general conduct concerns, open an issue only if it is appropriate to do so publicly. For sensitive matters, use the contact or security reporting process described in [SECURITY.md](SECURITY.md).
